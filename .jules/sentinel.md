## 2024-07-12 - [AppleScript Command Injection via Unescaped Backslashes]
**Vulnerability:** AppleScript command injection in `findSimulatorWindowBounds` in `lib/services/simulator_window_bounds.dart` where `deviceName` was interpolated into the script. The string `replaceAll('"', r'\"')` didn't escape backslashes first, allowing attackers to inject backslashes to escape the double quote escaping, and break out of the string context in AppleScript to execute arbitrary commands via `osascript`.
**Learning:** When constructing AppleScript commands with string interpolation in Dart (e.g., executed via `osascript`), only escaping double quotes isn't enough, as a user could submit a string ending in a backslash (e.g., `test\`), which would result in `test\"`, escaping the quote and keeping the string open, allowing command injection.
**Prevention:** Always escape backslashes before escaping double quotes (e.g., `.replaceAll(r'\', r'\\').replaceAll('"', r'\"')`) to prevent command injection vulnerabilities when interpolating strings into commands executed by the shell or osascript.
