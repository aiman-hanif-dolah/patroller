## 2025-02-18 - Prevent AppleScript Command Injection via Insufficient Escaping
**Vulnerability:** In `findSimulatorWindowBounds`, AppleScript constructed with string interpolation using `deviceName` failed to escape backslashes before escaping double quotes, leading to a potential command injection vulnerability. A user could input a string like `\"` which, after escaping only double quotes, becomes `\\"`, allowing them to break out of the string context in AppleScript.
**Learning:** When dynamically generating AppleScript commands or other executed code containing interpolated strings, simply escaping quotes (`"`) is insufficient. Backslashes (`\`) must be escaped *before* escaping the quotes to properly secure against command injection.
**Prevention:** Always use `.replaceAll(r'\', r'\\').replaceAll('"', r'\"')` when injecting Dart strings into AppleScript `osascript` string literals, or better yet, avoid string interpolation and use AppleScript parameter passing where possible.
